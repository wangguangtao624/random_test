# 随机性检测测试方案

## 一、参考标准文件

| 文件 | 说明 |
|---|---|
| **NIST SP 800-22 Rev.1a** | 美国标准，131页，定义了15项统计测试方法，显著性水平α=0.01 |
| **GB/T 32915-2016** | 中国国标，24页（扫描版），参考NIST但有中国特色要求，含单比特频率、扑克检测、游程检测、自相关检测等 |
| **实施方案.docx** | 已有的综合文档，包含两个标准的对比、15项测试详解、芯片TRNG设计建议、后处理方案、开源工具推荐 |

---

## 二、测试目标

验证随机数生成器（RNG/TRNG）输出的二元序列是否满足 NIST SP 800-22 和 GB/T 32915-2016 的随机性要求。

---

## 三、测试数据准备

| 项目 | 要求 |
|---|---|
| 数据来源 | 芯片TRNG硬件输出（或FPGA原型） |
| 序列长度 | 每组 ≥ 1,000,000 比特（1Mbit） |
| 序列组数 | ≥ 1000组（用于统计通过率） |
| 数据格式 | 二进制文件，每行一个比特序列 |
| 采集方式 | 多批次、不同时段采集，覆盖温度/电压变化 |

## 四、NIST SP 800-22 全部15项测试

| 序号 | 测试名称 | 检测目的 | 关键参数 |
|---|---|---|---|
| 1 | 频率测试（单比特）/ Frequency (Monobit) Test | 0和1的比例是否接近50:50 | — |
| 2 | 块内频率测试 / Frequency Test within a Block | 固定长度块内0/1比例 | 块大小M |
| 3 | 游程测试 / Runs Test | 连续相同位的数量 | — |
| 4 | 块内最长游程测试 / Test for Longest Run of Ones | 最长连续1的长度分布 | 块大小M |
| 5 | 二元矩阵秩测试 / Binary Matrix Rank Test | 矩阵秩分布，检测线性依赖 | 矩阵大小Q×Q |
| 6 | 离散傅里叶变换（频谱）测试 / DFT (Spectral) Test | 检测周期性特征 | — |
| 7 | 非重叠模板匹配测试 / Non-overlapping Template Matching Test | 特定模式出现频率 | 模板长度m |
| 8 | 重叠模板匹配测试 / Overlapping Template Matching Test | 重叠模式出现频率 | 模板长度m |
| 9 | Maurer通用统计测试 / Maurer's Universal Test | 检测序列是否可压缩 | 块大小L |
| 10 | 线性复杂度测试 / Linear Complexity Test | LFSR长度检测 | 块大小M |
| 11 | 序列测试 / Serial Test | 所有m位模式频率 | 模式长度m |
| 12 | 近似熵测试 / Approximate Entropy Test | 序列熵值检测 | 模式长度m |
| 13 | 累积和测试 / Cumulative Sums (Cusum) Test | 偏移的最大累积和 | — |
| 14 | 随机游程测试 / Random Excursions Test | 循环游程中各状态访问次数 | — |
| 15 | 随机游程变体测试 / Random Excursions Variant Test | 特定状态访问频率 | — |

**统一判定标准**：每项测试的 P-value ≥ 0.01 为通过。

## 五、GB/T 32915-2016 补充测试

在NIST 15项基础上，国标额外关注：

- **扑克检测**：检测m位子序列的出现频率
- **自相关检测**：检测序列与其移位版本的相关性
- **显著性水平**：α = 0.01（与NIST一致）
- **通过标准**：所有测试P-value均 ≥ 0.01

## 六、各测试项详细说明

### 6.1 频率测试（单比特）

- **目的**：检测整个序列中0和1的比例是否接近相等
- **方法**：将序列中的1映射为+1，0映射为-1，求和后计算统计量
- **判定**：P-value ≥ 0.01 通过

### 6.2 块内频率测试

- **目的**：检测固定长度块内0和1的比例是否均匀
- **方法**：将序列分成M位的块，计算每块中1的比例，再用卡方检验
- **判定**：P-value ≥ 0.01 通过

### 6.3 游程测试

- **目的**：检测序列中游程（连续相同比特）的数量是否符合随机预期
- **方法**：统计0→1和1→0的转换次数
- **判定**：P-value ≥ 0.01 通过

### 6.4 块内最长游程测试

- **目的**：检测块内最长连续1的长度分布是否随机
- **方法**：将序列分块，统计各块中最长1游程的频率分布
- **判定**：P-value ≥ 0.01 通过

### 6.5 二元矩阵秩测试

- **目的**：检测序列中二元矩阵的秩分布，发现线性依赖关系
- **方法**：构造Q×Q二元矩阵，计算秩，检查是否符合随机分布
- **判定**：P-value ≥ 0.01 通过

### 6.6 离散傅里叶变换（频谱）测试

- **目的**：检测序列中的周期性特征
- **方法**：对序列做FFT，检查频谱中超过阈值的峰值数量
- **判定**：P-value ≥ 0.01 通过

### 6.7 非重叠模板匹配测试

- **目的**：检测特定二元模板的出现频率
- **方法**：统计非重叠模板出现的次数，与期望值比较
- **判定**：P-value ≥ 0.01 通过

### 6.8 重叠模板匹配测试

- **目的**：检测重叠模式的出现频率
- **方法**：统计重叠模板出现的次数
- **判定**：P-value ≥ 0.01 通过

### 6.9 Maurer通用统计测试

- **目的**：检测序列是否可以被显著压缩（可压缩意味着非随机）
- **方法**：基于Lempel-Ziv压缩原理
- **判定**：P-value ≥ 0.01 通过

### 6.10 线性复杂度测试

- **目的**：检测序列是否可由较短的线性反馈移位寄存器(LFSR)生成
- **方法**：使用Berlekamp-Massey算法计算线性复杂度
- **判定**：P-value ≥ 0.01 通过

### 6.11 序列测试

- **目的**：检测所有可能的m位模式的出现频率是否均匀
- **方法**：统计2^m种模式的出现次数，用卡方检验
- **判定**：P-value ≥ 0.01 通过

### 6.12 近似熵测试

- **目的**：检测序列的熵值（信息量）是否符合随机预期
- **方法**：计算近似熵，与随机序列比较
- **判定**：P-value ≥ 0.01 通过

### 6.13 累积和测试

- **目的**：检测序列中偏移的最大累积和是否异常
- **方法**：计算累积和的正向和负向最大值
- **判定**：P-value ≥ 0.01 通过

### 6.14 随机游程测试

- **目的**：检测循环随机游程中各状态的访问次数
- **方法**：分析随机游程的8个状态（-4到-1和1到4）
- **判定**：P-value ≥ 0.01 通过

### 6.15 随机游程变体测试

- **目的**：检测随机游程中特定状态的访问次数
- **方法**：统计18个特定状态（-9到-1和1到9）的访问频率
- **判定**：P-value ≥ 0.01 通过

## 七、测试执行流程

```
阶段1: FPGA原型验证
  ├─ 采集TRNG原始输出
  ├─ 运行15项NIST测试 + GB/T补充测试
  ├─ 分析失败项，调整设计参数
  └─ 迭代直到通过率 ≥ 98%

阶段2: 流片后芯片验证
  ├─ 采集芯片实际输出（多批次）
  ├─ 完整运行全部测试
  ├─ 记录每项测试的P-value分布
  └─ 确保满足两个标准要求

阶段3: 量产测试
  ├─ 简化测试（运行核心测试项）
  ├─ 统计通过率
  └─ 筛选不良芯片
```

## 八、推荐后处理方案

原始TRNG输出通常有偏差，需后处理：

| 方案 | 优点 | 缺点 | 推荐度 |
|---|---|---|---|
| Von Neumann校正器 | 简单 | 吞吐量降50% | 基础方案 |
| **SHA-256/SM3哈希** | 效果好、输出均匀 | 需硬件哈希模块 | **推荐** |
| AES-CBC-MAC | 速度快 | 需AES硬件 | 备选 |
| LFSR后处理 | 简单 | 安全性较低 | 不推荐 |

**推荐架构**：TRNG + SHA-256/SM3 后处理

## 九、芯片级TRNG常见实现方式

| 方案 | 原理 | 优点 | 缺点 |
|---|---|---|---|
| 环形振荡器(RO) | 奇数个反相器构成振荡环，采样热噪声抖动 | 简单、面积小 | 速度慢、可能有偏差 |
| 亚稳态(Metastability) | 利用触发器的亚稳态，采样解析结果 | 随机性好 | 需要校准 |
| 热噪声放大 | 放大电阻热噪声，比较器采样 | 物理随机源 | 需要模拟电路 |
| 振荡器采样(OS) | 快时钟采样慢时钟 | 简单 | 随机性有限 |

## 十一、通过标准

- 所有15项NIST测试 P-value ≥ 0.01
- GB/T 32915-2016 全部测试通过
- 多组数据的总体通过率 ≥ 98%
- P-value分布应近似均匀（可用卡方检验二次验证）

## 十三、总结与建议

1. **设计阶段**：选择合适的TRNG架构（推荐RO+后处理方案），预留面积用于后处理模块，设计自测试接口
2. **验证阶段**：使用开源NIST测试套件，在FPGA上验证TRNG功能，迭代优化直到通过率≥98%
3. **流片后**：完整运行15项NIST测试，记录每项测试的P-value分布，确保满足GB/T 32915-2016要求
4. **量产阶段**：简化测试流程，统计通过率，持续监控质量
