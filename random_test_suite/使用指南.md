# CIS芯片TRNG随机性测试套件 - 使用指南

## 一、快速开始

### 1.1 环境准备

**系统要求：**
- Windows 10/11
- Python 3.7 或更高版本
- 至少 4GB 内存（处理大量数据时）

**安装步骤：**

```bash
# 方法1: 运行安装脚本
install.bat

# 方法2: 手动安装
pip install numpy scipy matplotlib
```

### 1.2 验证安装

```bash
python test_tool.py
```

如果看到"所有测试通过"，说明安装成功。

## 二、数据准备

### 2.1 数据来源

从CIS芯片的TRNG模块采集输出数据。采集方式：
- UART串口输出
- SPI接口读取
- 专用调试接口
- 芯片测试模式输出

### 2.2 数据格式要求

**支持的格式：**
- `.bin` - 二进制文件（推荐）
- `.txt` - 文本文件（每行一个比特或连续比特串）
- `.dat` - 二进制数据文件
- `.raw` - 原始二进制数据

**数据要求：**
- 每个文件至少包含 1,000,000 比特 (1Mbit)
- 建议准备 1000 个文件用于统计分析
- 文件命名建议：`sample_0001.bin`, `sample_0002.bin`, ...

### 2.3 数据采集建议

```
采集流程：
┌─────────────┐    ┌─────────────┐    ┌─────────────┐
│  CIS芯片    │───▶│  采集接口   │───▶│  数据文件   │
│  TRNG输出   │    │  UART/SPI   │    │  .bin/.txt  │
└─────────────┘    └─────────────┘    └─────────────┘

采集条件：
• 多批次采集（不同时间段）
• 覆盖温度变化（-40°C ~ 85°C）
• 覆盖电压变化（±10%）
• 不同工作模式
```

## 三、运行测试

### 3.1 测试单个文件

```bash
python main.py --file data/sample.bin
```

### 3.2 测试整个目录

```bash
python main.py --data-dir ./data
```

### 3.3 指定输出目录


```bash

python main.py --data-dir ./data --alpha 0.01

# 设置文件匹配模式
python main.py --data-dir ./data --pattern "*.bin"
```

## 四、理解测试结果

### 4.1 测试项说明

**NIST SP 800-22 测试（15项）：**

| 序号 | 测试名称 | 检测目的 |
|------|----------|----------|
| 1 | frequency_monobit | 0和1的比例是否接近50:50 |
| 2 | frequency_within_block | 块内0/1比例是否均匀 |
| 3 | runs | 游程数量是否符合随机预期 |
| 4 | longest_run_of_ones | 最长连续1的长度分布 |
| 5 | binary_matrix_rank | 矩阵秩分布，检测线性依赖 |
| 6 | dft_spectral | 检测周期性特征 |
| 7 | non_overlapping_template | 特定模式出现频率 |
| 8 | overlapping_template | 重叠模式出现频率 |
| 9 | maurers_universal | 检测序列是否可压缩 |
| 10 | linear_complexity | LFSR长度检测 |
| 11 | serial | 所有m位模式频率 |
| 12 | approximate_entropy | 序列熵值检测 |
| 13 | cumulative_sums | 偏移的最大累积和 |
| 14 | random_excursions | 循环游程中各状态访问次数 |
| 15 | random_excursions_variant | 特定状态访问频率 |

**GB/T 32915-2016 补充测试（3项）：**

| 序号 | 测试名称 | 检测目的 |
|------|----------|----------|
| 16 | poker_test_m4 | 4位子序列的出现频率 |
| 17 | poker_test_m8 | 8位子序列的出现频率 |
| 18 | autocorrelation_test | 序列与其移位版本的相关性 |
| 19 | binary_derivative_test | 逐级差分后0/1比例 |

### 4.2 通过标准

| 指标 | 要求 | 说明 |
|------|------|------|
| 单项测试P-value | ≥ 0.01 | 每项测试的最低要求 |
| 总体通过率 | ≥ 98% | 所有测试的平均通过率 |
| 测试序列长度 | ≥ 1Mbit/组 | 每组数据的最小长度 |
| 测试序列组数 | ≥ 1000组 | 统计意义的最小样本量 |

### 4.3 结果解读

**P-value 含义：**
- P-value ≥ 0.01：测试通过，序列表现出随机性
- P-value < 0.01：测试失败，序列可能存在非随机特征

**通过率含义：**
- 通过率 ≥ 98%：TRNG输出质量良好
- 通过率 < 98%：TRNG输出可能存在问题

## 六、常见问题

### 6.1 测试失败怎么办？

如果测试未通过，建议：

1. **检查数据质量**
   - 确认数据采集正确
   - 检查是否有数据损坏
   - 验证数据格式

2. **检查TRNG设计**
   - 环形振荡器(RO)设计是否合理
   - 后处理模块是否正常工作
   - 是否有足够的熵源

3. **调整测试参数**
   - 增加测试序列长度
   - 增加测试序列数量
   - 检查温度/电压条件

### 6.2 内存不足怎么办？

处理大量数据时可能内存不足：

```bash
# 减少同时处理的文件数量
python main.py --data-dir ./data --pattern "sample_0001.bin"

# 或分批处理
python main.py --data-dir ./data --pattern "sample_000*.bin"
python main.py --data-dir ./data --pattern "sample_001*.bin"
```

### 6.3 如何自定义测试参数？

编辑 `config.py` 文件可以修改：
- 显著性水平
- 测试参数
- 通过标准

## 七、高级用法

### 7.1 编程接口

```python
from core.nist_tests import NISTTestSuite
from core.gbt_tests import GBTTestSuite
from utils.data_loader import DataLoader

# 加载数据
loader = DataLoader()
bits = loader.load_from_file('data/sample.bin')

# 运行NIST测试
nist_suite = NISTTestSuite()
nist_results = nist_suite.run_all_tests(bits)

# 运行GB/T测试
gbt_suite = GBTTestSuite()
gbt_results = gbt_suite.run_all_tests(bits)

# 打印结果
for test_name, p_value in {**nist_results, **gbt_results}.items():
    print(f"{test_name}: {p_value:.6f}")
```

### 7.2 批量处理脚本

```python
import os
from main import CISChipRandomnessTest
from utils.report_generator import ReportGenerator

# 创建测试套件
tester = CISChipRandomnessTest()

# 处理多个目录
directories = ['data/batch1', 'data/batch2', 'data/batch3']

for directory in directories:
    print(f"\n处理目录: {directory}")
    results = tester.run_directory_test(directory)

    # 生成报告
    report_gen = ReportGenerator(f'./reports/{os.path.basename(directory)}')
    report_gen.generate_html_report(results)
```

### 7.3 集成到CI/CD

```yaml
# .github/workflows/randomness_test.yml
name: TRNG Randomness Test

on: [push, pull_request]

jobs:
  test:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v2
      - name: Set up Python
        uses: actions/setup-python@v2
        with:
          python-version: '3.9'
      - name: Install dependencies
        run: pip install numpy scipy matplotlib
      - name: Run tests
        run: python main.py --data-dir ./test_data
```
