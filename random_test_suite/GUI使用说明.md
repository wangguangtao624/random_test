# CIS芯片TRNG随机性测试平台 - GUI使用说明

## 启动平台

### 方式1：命令行启动
```bash
python gui_app.py
```

### 方式2：双击启动脚本
```
启动测试平台.bat
```

## 使用步骤

### 第一步：选择测试项目

在左侧面板中勾选要运行的测试项目：

**NIST SP 800-22 测试项 (15项)：**
- 单比特频数检测 - 检测0和1的比例是否接近50:50
- 块内频数检测 - 检测块内0/1比例是否均匀
- 游程检测 - 检测游程数量是否符合随机预期
- 块内最长游程检测 - 检测最长连续1的长度分布
- 二元矩阵秩检测 - 检测矩阵秩分布，发现线性依赖
- 离散傅里叶检测 - 检测周期性特征
- 非重叠模板匹配检测 - 检测特定模式出现频率
- 重叠模板匹配检测 - 检测重叠模式出现频率
- Maurer通用统计检测 - 检测序列是否可压缩
- 线性复杂度检测 - 检测LFSR长度
- 序列检测 - 检测所有m位模式频率
- 近似熵检测 - 检测序列熵值
- 累加和检测 - 检测偏移的最大累积和
- 随机游程检测 - 检测循环游程中各状态访问次数
- 随机游程变体检测 - 检测特定状态访问频率

**GB/T 32915-2016 补充测试项 (4项)：**
- 扑克检测 (m=4) - 检测4位子序列的出现频率
- 扑克检测 (m=8) - 检测8位子序列的出现频率
- 自相关检测 - 检测序列与其移位版本的相关性
- 二元推导检测 - 检测逐级差分后0/1比例

**快捷按钮：**
- [全选] - 选择所有测试项
- [反选] - 反转当前选择
- [NIST全选] - 只选择NIST测试项
- [GB/T全选] - 只选择GB/T测试项

**查看测试详情：**
点击每个测试项旁边的 [详情] 按钮，可以查看：
- 测试名称（中英文）
- 所属标准
- 测试目的
- 检测方法
- 详细说明
- 通过标准

### 第二步：选择测试数据

在右上方面板中选择测试数据文件：

**选择方式：**
- [选择单个文件] - 选择单个测试数据文件
- [选择目录] - 选择包含多个数据文件的目录
- [清空] - 清空已选择的文件列表

**支持的文件格式：**
- .bin - 二进制文件（推荐）
- .txt - 文本文件
- .dat - 数据文件
- .raw - 原始数据

**数据要求：**
- 每个文件至少 1,000,000 比特 (1Mbit)
- 建议准备 1000 个文件用于统计分析

### 第三步：运行测试

点击底部的 [开始测试] 按钮开始测试。

**测试过程中：**
- 右下方面板会显示实时日志
- 进度条会显示当前进度
- 状态栏会显示当前状态

**测试完成后：**
- 切换到 [测试结果] 标签页查看结果
- 结果会显示每项测试的通过率和状态

## 测试项详细说明

### NIST SP 800-22 测试项

#### 1. 单比特频数检测 (Frequency Monobit Test)
- **目的**：检测整个序列中0和1的比例是否接近相等
- **方法**：将序列中的1映射为+1，0映射为-1，求和后计算统计量
- **用途**：验证TRNG输出的比特分布是否均匀，是最基本的随机性测试

#### 2. 块内频数检测 (Frequency Test within a Block)
- **目的**：检测固定长度块内0和1的比例是否均匀
- **方法**：将序列分成M位的块，计算每块中1的比例，再用卡方检验
- **用途**：检测序列在局部范围内是否保持均匀分布

#### 3. 游程检测 (Runs Test)
- **目的**：检测序列中游程（连续相同比特）的数量是否符合随机预期
- **方法**：统计0→1和1→0的转换次数
- **用途**：验证序列中比特交替的频率是否符合随机特性

#### 4. 块内最长游程检测 (Test for Longest Run of Ones)
- **目的**：检测块内最长连续1的长度分布是否随机
- **方法**：将序列分块，统计各块中最长1游程的频率分布
- **用途**：检测序列中是否存在异常长的连续相同比特

#### 5. 二元矩阵秩检测 (Binary Matrix Rank Test)
- **目的**：检测序列中二元矩阵的秩分布，发现线性依赖关系
- **方法**：构造Q×Q二元矩阵，计算秩，检查是否符合随机分布
- **用途**：检测序列中的线性依赖关系，验证序列的不可预测性

#### 6. 离散傅里叶检测 (DFT Spectral Test)
- **目的**：检测序列中的周期性特征
- **方法**：对序列做FFT，检查频谱中超过阈值的峰值数量
- **用途**：发现序列中隐藏的周期性模式或重复特征

#### 7. 非重叠模板匹配检测 (Non-overlapping Template Matching Test)
- **目的**：检测特定二元模板的出现频率
- **方法**：统计非重叠模板出现的次数，与期望值比较
- **用途**：检测序列中是否存在特定的重复模式

#### 8. 重叠模板匹配检测 (Overlapping Template Matching Test)
- **目的**：检测重叠模式的出现频率
- **方法**：统计重叠模板出现的次数
- **用途**：检测序列中重叠模式的分布是否符合随机预期

#### 9. Maurer通用统计检测 (Maurer's Universal Statistical Test)
- **目的**：检测序列是否可以被显著压缩
- **方法**：基于Lempel-Ziv压缩原理，计算序列的可压缩性
- **用途**：验证序列的信息熵是否足够高，不可被有效压缩

#### 10. 线性复杂度检测 (Linear Complexity Test)
- **目的**：检测序列是否可由较短的线性反馈移位寄存器(LFSR)生成
- **方法**：使用Berlekamp-Massey算法计算线性复杂度
- **用途**：验证序列的不可预测性，防止被LFSR预测

#### 11. 序列检测 (Serial Test)
- **目的**：检测所有可能的m位模式的出现频率是否均匀
- **方法**：统计2^m种模式的出现次数，用卡方检验
- **用途**：验证所有可能的比特组合是否等概率出现

#### 12. 近似熵检测 (Approximate Entropy Test)
- **目的**：检测序列的熵值（信息量）是否符合随机预期
- **方法**：计算近似熵，与随机序列比较
- **用途**：衡量序列的复杂度和不可预测性

#### 13. 累加和检测 (Cumulative Sums Test)
- **目的**：检测序列中偏移的最大累积和是否异常
- **方法**：计算累积和的正向和负向最大值
- **用途**：检测序列中是否存在系统性偏移

#### 14. 随机游程检测 (Random Excursions Test)
- **目的**：检测循环随机游程中各状态的访问次数
- **方法**：分析随机游程的8个状态（-4到-1和1到4）
- **用途**：验证随机游程的统计特性是否符合预期

#### 15. 随机游程变体检测 (Random Excursions Variant Test)
- **目的**：检测随机游程中特定状态的访问次数
- **方法**：统计18个特定状态（-9到-1和1到9）的访问频率
- **用途**：更全面地验证随机游程的统计特性

### GB/T 32915-2016 补充测试项

#### 16. 扑克检测 (m=4) (Poker Test)
- **目的**：检测4位子序列的出现频率是否均匀
- **方法**：将序列分为4比特的组，统计16种可能取值的出现次数，用卡方检验
- **用途**：验证短模式的分布是否符合随机预期

#### 17. 扑克检测 (m=8) (Poker Test)
- **目的**：检测8位子序列的出现频率是否均匀
- **方法**：将序列分为8比特的组，统计256种可能取值的出现次数，用卡方检验
- **用途**：验证较长模式的分布是否符合随机预期

#### 18. 自相关检测 (Autocorrelation Test)
- **目的**：检测序列与其移位版本的相关性
- **方法**：计算序列与不同位移版本的异或结果，统计0的个数
- **用途**：检测序列中是否存在周期性或相关性

#### 19. 二元推导检测 (Binary Derivative Test)
- **目的**：对序列逐级进行差分（异或相邻比特），检查各级推导序列的均衡性
- **方法**：计算差分序列，检查各级推导序列中0和1的比例
- **用途**：检测序列的高阶统计特性

## 通过标准

| 指标 | 要求 | 说明 |
|------|------|------|
| 单项测试P-value | ≥ 0.01 | 每项测试的最低要求 |
| 总体通过率 | ≥ 98% | 所有测试的平均通过率 |
| 测试序列长度 | ≥ 1Mbit/组 | 每组数据的最小长度 |
| 测试序列组数 | ≥ 1000组 | 统计意义的最小样本量 |

## 常见问题

### Q: 如何查看测试项的详细说明？
A: 点击每个测试项旁边的 [详情] 按钮，可以查看完整的测试说明。

### Q: 如何只运行部分测试项？
A: 在左侧面板中取消勾选不需要的测试项，或使用快捷按钮进行选择。

### Q: 测试数据格式有什么要求？
A: 支持 .bin、.txt、.dat、.raw 格式，每个文件至少 1,000,000 比特。

### Q: 测试需要多长时间？
A: 取决于数据文件数量和大小，单个1Mbit文件约需1-5秒。

### Q: 如何查看测试报告？
A: 测试完成后点击 [生成报告] 按钮，报告会保存到指定目录。
